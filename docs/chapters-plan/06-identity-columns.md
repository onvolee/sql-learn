## 任务清单

### Task 1 — 写 `schema.ts`

三张表分别用三种 ID 策略：
- `t_serial(id serial PK, name text)` — 经典 SERIAL（兼容写法）
- `t_identity(id integer GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY, name text)` — SQL 标准
- `t_uuid(id uuid PK DEFAULT gen_random_uuid(), name text)` — UUID 主键

注：drizzle 对 IDENTITY 的支持视版本而定；若 DSL 不支持，schema.ts 仅声明能表达的部分，其余在 seed.sql 里补 `ALTER TABLE` 添加 IDENTITY。

### Task 2 — 写 `seed.sql`

每张表 3 行。SERIAL / IDENTITY 不显式给 id（让序列推进），UUID 显式给固定值便于 example 引用。全部 `ON CONFLICT DO NOTHING`。

### Task 3 — 写 Section 1：SERIAL — 经典自增

- **定义段要点**：`SERIAL` 是「`integer` + 关联一个 SEQUENCE + 默认值」的语法糖 / 列实际类型还是 integer / 兼容历史代码，新代码推荐 IDENTITY
- **语法骨架**：ASCII，`CREATE TABLE ... (id serial PRIMARY KEY)` + 等价展开
- **mermaid**：不加
- **Examples**：
  - `serial-insert-default` — drizzle DSL，不传 id，看自增；`support: 'full'`
  - `serial-currval` — `SELECT currval(pg_get_serial_sequence('t_serial', 'id'))`；`support: 'partial'`

### Task 4 — 写 Section 2：GENERATED AS IDENTITY — 标准化写法

- **定义段要点**：SQL 标准的自增写法（PG 10+） / `BY DEFAULT` 允许手动覆盖、`ALWAYS` 强制系统生成 / 比 SERIAL 更干净（权限、迁移更友好）
- **语法骨架**：ASCII，`GENERATED { BY DEFAULT | ALWAYS } AS IDENTITY` 对比
- **mermaid**：不加
- **Examples**：
  - `identity-default-insert` — 不传 id 插入；`support: 'partial'`
  - `identity-explicit-override` — `BY DEFAULT` 模式下手动给 id（指定一个不冲突的大数 如 `id=10001`）；`support: 'partial'`
  - `identity-always-rejects` — 演示 `GENERATED ALWAYS` 强制覆盖需要 `OVERRIDING SYSTEM VALUE`（不加直接插会报错）；`support: 'partial'`（example 中临时建一张 ALWAYS 表并清理）

### Task 5 — 写 Section 3：UUID 主键

- **定义段要点**：`uuid` 是 128 位标识 / `gen_random_uuid()` 内建（PG 13+，无需扩展） / 分布式生成无需中心 / 缺点：体积大、有序性差
- **语法骨架**：ASCII，`id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
- **mermaid**：不加
- **Examples**：
  - `uuid-default-insert` — drizzle DSL，不传 id 插入；`support: 'partial'`
  - `uuid-explicit` — 显式给 UUID 字面量 `'00000000-0000-0000-0000-000000000001'::uuid` 插入；`support: 'partial'`

### Task 6 — 写 Section 4：SEQUENCE — 底层对象

- **定义段要点**：SEQUENCE 是 PG 的独立对象，专门「产生递增整数」 / SERIAL / IDENTITY 都基于它 / `nextval()` 取下一个、`currval()` 取当前会话的、`setval()` 改值
- **语法骨架**：ASCII，`CREATE SEQUENCE <name>` + `nextval / currval / setval` 三函数
- **mermaid**：不加
- **Examples**：
  - `seq-create-use` — `CREATE SEQUENCE IF NOT EXISTS s_demo START 100`；接着 `SELECT nextval('s_demo')`；`support: 'partial'`
  - `seq-currval` — 同一 example 内 `nextval` 后 `currval`；`support: 'partial'`
  - `seq-inspect` — 查 `information_schema.sequences` 看本 schema 所有序列；`support: 'partial'`
